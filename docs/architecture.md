# Architecture

This monorepo contains four packages that together produce a Chrome Extension (Manifest V3).

## Packages

- `@chrome-ext/popup` — React application built with Vite.
  - Outputs: `dist/index.html`, `dist/popup.js`, `dist/assets/*`
- `@chrome-ext/content-script` — Page-injected script built with Rollup.
  - Outputs: `dist/content-script.js` (IIFE), plus ESM/CJS builds for reuse.
- `@chrome-ext/background` — Background service worker built with Rollup.
  - Outputs: `dist/index.mjs` and `dist/index.cjs` (the assembler renames to `background.js`).
- `@chrome-ext/extension` — Assembler that creates the final extension folder.
  - Script: `src/index.ts`
  - Produces: `dist/manifest.json`, `background.js`, `content-script.js`, `popup.html`, `assets/*`, `icons/*`

## Build pipeline

1. Each package builds to its own `dist/` folder
2. The assembler (`@chrome-ext/extension`) reads built outputs from `node_modules/@chrome-ext/*/dist` (pnpm workspace symlinks)
3. The assembler copies/renames assets into `packages/extension/dist/` and generates `manifest.json`

## Manifest

The manifest is generated by `packages/extension/src/index.ts` with sane defaults:

- Manifest V3
- `action.default_popup` → `popup.html`
- `background.service_worker` → `background.js`
- `content_scripts[0].js` → `content-script.js`
- Common permissions: `storage`, `activeTab`, `tabs`, `cookies`, `alarms`, `contextMenus`

Adjust matches/permissions and other settings in `src/index.ts` as needed.

## Bundlers

- Rollup for background and content script with TypeScript plugin
- Vite for popup (configures output filenames to `popup.js` etc.)

## Monorepo tooling

- pnpm workspaces for dependency management
- Turborepo to orchestrate `build`, `dev`, `type-check`, `lint`, and `clean` across packages

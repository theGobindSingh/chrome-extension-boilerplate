# Architecture

This monorepo contains six packages that together produce a Chrome Extension (Manifest V3).

## Packages

- `@chrome-ext/popup` — React application built with Vite.
  - Outputs: `dist/index.html`, `dist/popup.js`, `dist/assets/*`
- `@chrome-ext/content-script` — Page-injected script built with Rollup.
  - Outputs: `dist/content-script.js` (IIFE), plus ESM/CJS builds for reuse.
  - Runs in an isolated JS world (standard content-script sandboxing) with access to `chrome.*` APIs.
- `@chrome-ext/page-bridge` — Script built with Rollup, output as a single IIFE `dist/page-bridge.js`.
  - Not a content script — it's injected by `@chrome-ext/content-script` as a `<script src="chrome-extension://.../page-bridge.js">` tag, which runs it in the page's own JS context ("MAIN world"). This is the standard way to reach page-defined globals from an extension, and it's why the manifest must list it under `web_accessible_resources`.
- `@chrome-ext/styles` — SCSS + PostCSS (autoprefixer) pipeline, run via a small `tsx` build script.
  - Output: `dist/content-script.css`, declared in the manifest's `content_scripts.css`. Because the browser injects it the same privileged way it injects `content-script.js`, it isn't subject to the host page's CSP the way inline styles or JS-created `<style>` elements would be.
- `@chrome-ext/background` — Background service worker built with Rollup.
  - Outputs: `dist/index.mjs` and `dist/index.cjs` (the assembler renames to `background.js`).
- `@chrome-ext/extension` — Assembler that creates the final extension folder.
  - Script: `src/index.ts`
  - Produces: `dist/manifest.json`, `background.js`, `content-script.js`, `content-script.css`, `page-bridge.js`, `popup.html`, `assets/*`, `icons/*`

## Build pipeline

1. Each package builds to its own `dist/` folder
2. The assembler (`@chrome-ext/extension`) reads built outputs from `node_modules/@chrome-ext/*/dist` (pnpm workspace symlinks)
3. The assembler copies/renames assets into `packages/extension/dist/` and generates `manifest.json`
4. `@chrome-ext/background`, `@chrome-ext/content-script`, and `@chrome-ext/page-bridge` are discovered generically (each has a `.js` main file); `@chrome-ext/popup` (HTML + assets) and `@chrome-ext/styles` (CSS) are copied via their own dedicated steps since their outputs aren't a single JS main file

## Manifest

The manifest is generated by `packages/extension/src/index.ts` with sane defaults:

- Manifest V3
- `action.default_popup` → `popup.html`
- `background.service_worker` → `background.js`
- `content_scripts[0].js` → `content-script.js`
- `content_scripts[0].css` → `content-script.css`
- `web_accessible_resources[0].resources` → `["page-bridge.js"]`
- Common permissions: `storage`, `activeTab`, `tabs`, `cookies`, `alarms`, `contextMenus`

Adjust matches/permissions and other settings in `src/index.ts` as needed.

## Bundlers

- Rollup for background, content-script, and page-bridge with TypeScript plugin
- Vite for popup (configures output filenames to `popup.js` etc.)
- Sass + PostCSS (autoprefixer) for styles, run via a small `tsx` script

## Monorepo tooling

- pnpm workspaces for dependency management
- Turborepo to orchestrate `build`, `dev`, `type-check`, `lint`, and `clean` across packages
